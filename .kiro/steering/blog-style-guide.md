# 博客内容风格指南

本项目是一个基于 Astro 的技术博客。创建或编辑博客文章时，请遵循以下规范。

## 文件结构

博客文章存放在 `src/content/blog/` 目录下，每篇文章使用独立文件夹：

```
src/content/blog/
└── {slug}/
    ├── index.md      # 文章内容
    └── *.png/jpg     # 文章配图（可选）
```

- `{slug}` 使用 kebab-case 命名，如 `typescript-tips`、`ai-ide-git-to-blog`
- 配图放在文章同级目录，使用相对路径引用：`![描述](./image.png)`

## Frontmatter 格式

```yaml
---
title: "文章标题"
description: "一句话描述文章内容，用于 SEO 和列表展示"
date: 2025-12-09T10:00+08:00
updatedDate: 2025-12-09T16:00+08:00  # 可选，有重大更新时添加
tags: ["标签1", "标签2"]
draft: false      # 可选，草稿状态
pinned: 1         # 可选，置顶权重，数字越大越靠前
cover:            # 可选，封面图
  url: "./cover.png"
  alt: "封面描述"
---
```

### 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| title | ✅ | 文章标题，建议 15-30 字 |
| description | ✅ | 简短描述，建议 50-100 字 |
| date | ✅ | 发布时间，ISO 格式带时区 |
| updatedDate | ❌ | 更新时间，仅重大修改时添加 |
| tags | ❌ | 标签数组，建议 2-4 个 |
| draft | ❌ | 草稿状态，默认 false |
| pinned | ❌ | 置顶权重 |
| cover | ❌ | 封面图配置 |

## 内容风格

### 语气与表达

- 专业但不僵硬，像和同事分享经验
- 使用「我」或「我们」拉近距离
- 避免过度使用感叹号和夸张用语

### 结构建议

1. **开篇**：简短引入问题或背景（2-3 段）
2. **正文**：分章节展开，每节有明确主题
3. **结尾**：总结要点或给出行动建议

### Markdown 规范

- 使用 `---` 分隔主要章节
- 使用 `>` 引用块强调关键概念
- 代码块指定语言：\`\`\`typescript
- 表格用于对比或列举结构化信息
- 图片添加 alt 描述和说明文字

### 示例

```markdown
## 章节标题

简短的章节引言。

> 关键概念或重要提示

### 子章节

正文内容...

| 对比项 | 选项 A | 选项 B |
|--------|--------|--------|
| 特点 1 | ✅ | ❌ |

![示意图](./diagram.png)
*图片说明文字*
```

## 常用标签

参考现有标签保持一致性：

- 技术栈：`TypeScript`、`前端`、`Node.js`
- 主题：`AI`、`效率工具`、`写作`、`设计`
- 类型：`教程`、`技术博客`、`开发工具`

## 标题命名建议

好的标题应该：
- 明确传达文章价值
- 包含关键词便于搜索
- 避免过于宽泛或点击诱导

示例：
- ✅ "TypeScript 实用技巧分享"
- ✅ "用 AI 编程工具将代码提交自动转化为技术博客"
- ❌ "修改了 Auth.ts"
- ❌ "你绝对想不到的 10 个技巧"
